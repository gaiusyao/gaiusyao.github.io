产品汪的工具箱

——一只还很Naïve的产品汪，很惭愧，做了一些微小的工作。

  最近在复盘自己三个月的产品工作，正好一直热衷于体验和分析各种各样的产品，那复盘的第一步就对自己经常使用的一些工具做一个简单的梳理。
  先简要列出在自己工具箱中各类常用物件：

  1. 纸笔尺
  2. ProcessOn、Xmind
  3. 有道云笔记、石墨文档
  4. Axure、Mockplus
  5. Worktile
  6. Affinity Designer、Affinity Photo
  7. Adobe illustrator、Adobe Photoshop
  8. R Studio
  9. VS Code、Sublime Text

// 不包括office、git、chrome等基本工具

1.1 入选原则

  工具箱内物件的选择基本遵循以下三条原则：

  • 能为解决问题的方式带来更可靠、更高效、更惊艳的变化
  • 从实际需要出发，不片面追求工具的强大、逼格
  • 正版优先，尊重自己和他人的付出

1.1.1 解决问题

  工具是用来解决问题的，任何工具的选用都要围绕着问题能否得到解决来展开，这是首要原则。但能解决问题只是最基本的需求，工具还要能为解决问题的方式带来更可靠、更高效、更惊艳的变化，也就是为问题的解决提供更优解。

1.1.2 从实际出发

  过去有段时间，片面追求具有强大功能和酷炫效果的工具（换句话说就是能装逼），在电脑里装了很多、很杂的工具，然而并没有什么用。一方面没有那么多精力学习那么多工具，另一方面很多工具当时根本就用不上。如果说还有一点什么原因，就是当时那台电脑的性能较差，这个对工具的使用有很大的限制。因此，要一切从实际出发，实事求是，正确地选用适合的工具。

1.1.3 支持正版

  对于正版软件，你问我支持不支持，我是支持的。我就明确地给你告诉这一点。当然，由于种种原因（没钱），也用过很多“破解版”。 // 目前除了Axure和Adobe全家桶以外，工具箱的物件都已是正版

1.2 常用工具简介

1.2.1 纸笔尺

  最基本，也是最方便的工具，现在很多事情都实现了无纸化，但直接在纸上写/画，仍是最快捷的一种形式，尤其是在自己理清思绪、和团队沟通的过程中，可以快速的记录、整理思想的火花。

1.2.2 ProcessOn、Xmind

  ProcessOn很吼很强大，支持思维导图、流程图、UML等图形的绘制，也支持小组协作，可免费使用（建议升级为个人版）。 // 传送门：https://www.processon.com   Xmind就不用多说了吧，用于绘制思维导图和树状结构图。

1.2.3 有道云笔记、石墨文档

  有道云笔记使用的人也不少，主要用来做一些个人的记录。另，推荐使用Markdown编写。   石墨文档主要用于写需求文档，它支持在线的文档/表格协作，和多人的编辑、评论，对于团队协作来说十分便利。   以上两者都支持免费使用。 // 传送门：https://shimo.im

1.2.4 Axure、Mockplus

  Axure可以说是原型界执牛耳者，功能强大且完善，而且按照向下兼容的原则，它基本可以满足从高保真原型到基本线框图的所有要求。很吼很强大，就是太贵了...   对于像本人一样囊中羞涩，但又希望“骄傲用正版”的诸君，就推荐使用Mockplus啦。号称可以在十分钟内做出一个中保真的原型，而且价格十分亲民，就是某些方面还比不上Axure。

1.2.5 Worktile

  Worktile是一个团队协作项目管理工具，我司正在使用，体验还不错，想了解更多的团队管理工具可以看这篇文章：http://www.pmcaff.com/article/index/802628657245312?from=search // 传送门：https://worktile.com/

1.2.6 Affinity Designer、Affinity Photo

重点安利对象，前者是“是最快、最流畅、最精确的矢量图形设计软件”，可以胜任UI\UX，甚至原型设计的工作。而后者则是一款专业级的图形编辑软件，具有卓越的性能和令人惊叹的特性。值得一提的是，Affinity对标的是Adobe的Ai和Ps，很多地方（如快捷键位）有相似之处，对于有过Ai和Ps基础的人来说，很快就可以上手。同时，相较于Adobe，其价格也十分亲民，单个软件仅售￥328。 // 传送门：https://affinity.serif.com/zh-cn/

1.2.7 Adobe illustrator、Adobe Photoshop

  这两个不用介绍了吧，正努力成长为可以负担起Adobe全家桶的汪...

1.2.8 R Studio

  目前正在学习使用R进行数据分析和可视化，而R Studio就是R的一个IDE。

1.2.9 VS Code、Sublime Text

  写代码用（主要是前端）。

1.3 小结

  以上便是对本人工具箱中常用物件的简要介绍，在平常的产品工作中，主要用到的是前五个，其中：

  • 纸笔尺用于草稿/草图、用户故事地图以及沟通记录

  • ProcessOn、Xmind主要用于绘制思维导图、树状结构图和流程图

  • 有道云笔记、石墨文档分别用于个人记录和团队需求文档协作

  • Axure、Mockplus主要用于构建原型（现多为Mockplus）

  • Worktile则是用于团队协作和项目管理

      另外 Affinity Designer、Affinity Photo和 Adobe illustrator、Adobe Photoshop在平常的工作中使用的较少，更多是个人的一种兴趣。尤其是在心情不好的时候，做做图或修修片，便可尽释胸中块垒。而 R Studio和 VS Code、Sublime Text则是为了自学技术，学技术的原因将放在下一篇，这也是为什么这两点说的比较简略的原因。

      其实工具不重要，选择合适的工具解决问题才重要。产品的使命始终是发现问题和解决问题，并不断地为解决问题的方式带来更可靠、更高效、更惊艳的变化。因此，产品汪们必须让工具服务于目的，而不是陷入唯工具论的误区。另一方面，“工欲善其事必先利其器”，一套合适的工具也能带来不错的收益。
